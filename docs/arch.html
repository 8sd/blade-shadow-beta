Shadow tweaks

The Linux community for Shadow.

Arch Linux

The application works well on Arch Linux but needs several tweaks to work perfectly. We will use the AUR entry made by agentcobra#6142 because it simplifies a lot the installation.

First things first, several dependencies need to be installed:
sudo pacman --needed -S gconf libnotify libappindicator libxtst nss libcurl-compat libva libdrm freetype2 libbsd json-c opus sdl gcc7-libs ttf-dejavu.

As the application was originally designed for Ubuntu, some fixes in the installed dependencies need to be done. Follow the next commands: 

echo "Correct Libjson-c"
sudo ln -f /usr/lib/libjson-c.so /usr/lib/libjson-c.so.3

echo "Correct Libubsan"
sudo cp /usr/lib/gcc/x86_64-pc-linux-gnu/7.3.1/libubsan.so.0 /usr/lib/

echo "Fix the font issue"
sudo ln -sf /usr/share/fonts/TTF /usr/share/fonts/truetype

Then, we properly install the Shadow application. It depends on your favorite AUR installer:

  • Yay: yay -S shadow-beta
  • Yaourt: yaourt -S shadow-beta
  • Packer: packer -s shadow-beta
  • Pacaur: pacaur -s shadow-beta

Shadow does not work on Wayland, so if echo $XDG_SESSION_TYPE does not return x11, log out, click on the engine weel and select "Xorg" then log in again. It will look exactly the same but it will actually run on Xorg.
If you can't find the Xorg version of your desktop environement, please follow the Arch topic about it.

You will need also to add your user to the "input" group by doing: sudo gpasswd -a $USER input. You will need to reboot after this step.

For all distribution, you may experience an issue with the VA-API which is involved in the decoding process. Please follow the tutorial we made right here.
